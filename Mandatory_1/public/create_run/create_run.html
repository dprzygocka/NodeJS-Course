Tools

EXPRESS (node package/module)

  • Tooling for HTTP servers, solution for single page applications, web sites, hybrids, or public HTTP APIs.

  • Installation: npm install express

NODEMON

  • Tool which automatically restarting the node application when file changes in the directory are detected.

  • To use it type command: nodemon.cmd < file name.js > (.cmd only necessary for windows users).

  • Installation: npm install -g nodemon

Basic Project Strucutre

PACKAGE.JSON contains:

  • dependencies

  • scripts

  • meta information

Example:
{ "dependencies": {
"express": "^4.17.1" } }

PACKAGE-LOCK.JSON:

  • contains meta information of what actually was installed on the machine that user is working on

  • created by command npm install < library name >

NODE MODULES:

  • folder in which packages and the their external libraries (that they rely on, defined in their internal package.json) are defined

  • ignored by git: person who gets the project can run npm install and based on package.json create their own node module folder

Create & Run

  1. Inside project folder create file package.json:

    • cd < folder name > //change direcotry using cd to locate yourself in project folder

    • place curly brackets n the package.json file

    • npm install (library name) //installs express in package.json

  2. Inside project folder create entry file called app.js where:

    • Import library that we are using (in this case is express):

      Example:
      const express = require("express");

    • Instantiate the server:

      Example:
      const app = express();

    • Define the port on which the server will listen.
      App.listen accepts two arguments where first one is port number and second is callback function which handles potential error

      Example:
      app.listen(8080, e => e ? console.log(e) : 'server is running on: ',8080);

    • Define the routes:

      Example:
      app.get( < endpoint >, (req,res) => { res.send()})

  3. Run project by typing in command line:

    • nodemon/node < file name > // nodemon.cmd on windows

    • after the command above node modules appears in the project structure
