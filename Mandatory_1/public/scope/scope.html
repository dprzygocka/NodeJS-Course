Scope & Variables

Scope

Global scope

  • Variables which are declared outside of any functions, can be accessed from anywhere in the program.

  • Variables declared inside a block can be access from outside of the block.

  • Redeclaring a variable inside a block causes the variable to change outside of the block.

  • Declaring variable with var or let ouside of a block will result with global scope for both of them.

  • In HTML global scope is the window object. Global variables declared with var belong to windows object where defined with let does not.

Function scope

  • Variables which are declared inside a function, can be accessed from inside of the given function.

  • Variables declared inside a block can NOT be access from outside of the block.

  • Redeclaring a variable inside a block causes the variable to change outside of the block.

  • Declaring variable with var or let inside function will result with function scope for both of them.

Loop scope

Example:
let/var i = 2
for (let/var i = 0, i < 10, i++){}

  • Declaring variable i using var in loop scope causes change of the variable outside of the loop

  • Declaring variable i using let in loop scope causes i to be visible only within the loop scope

Variables

Containers for storing data values.

Variable "let"

  • Defines variable in restricted scope

  • Provides block scope

  • Redeclaring a let variable with let, in the same scope, or in the same block, is not allowed

Variable "const"

  • Defines variables which can not be changed

  • Provides block scope

  • Defines a constant reference to a value which mean that we cannot change constant primitive values, but we can change the properties of constant objects

Variable "var"

  • Defines variable in file/global scope

  • Redeclaring is allowed anywhere in the program

  • Defines a constant reference to a value which mean that we cannot change constant primitive values, but we can change the properties of constant objects

Redeclaring a var variable with let or const(or other way around), in the same scope, or in the same block, is not allowed.
