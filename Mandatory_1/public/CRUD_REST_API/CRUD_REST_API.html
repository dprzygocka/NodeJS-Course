CRUD REST APIs

RESTful API

  • Application programming interface (API or web API) that conforms to the constraints of REST(representational state transfer).

  • Architectural style and allows for interaction with RESTful web services.

  • When a client request is made via a RESTful API, it transfers a representation of the state of the resource to the endpoint.

  • Information is delivered in different formats via HTTP: JSON (Javascript Object Notation), HTML, XLT, Python, PHP, or plain text.

CRUD REST API

REST uses the HTTP protocol’s request types (POST, GET, PUT, and DELETE) to allow users to Create, Read, Update, and Delete (CRUD) via an API.

GET request:

  • Used to query resources:

    • Server has security feature that allows to send only HTML files.
      In order to serve static files we need to include workaround.

      Example:
      app.use(express.static("public"));

  • Data are send via URL

    Example via route parameter:
    localhost:8080/variable (to retrive use req.params)
    Examplevia request parameter:
    localhost:8080/users? name = value & name = value (to retrive use req.query)
GET Request Example
app.get(“/path”, (req, res) => {
res.send() //send back response to user
})
GET request with identifier Example
app.get(“/path/:identifier”, (req, res) => {
req.params.identidfier // retives identifier send in url
})

POST request:

  • Used to create resources

  • Data are send in the body of the request

  • To handle POST in express we need middleware module called body-parser.
    It extracts the entire body portion of an incoming request stream and makes it accessible on req.body.

    Application in code example:
    app.use(express.json());
POST Request Example
app.post(“/path”, (req,res) => {
req.body() // to extract body of the request
//do somthing with the body
res.send()
})

PUT request:

  • Used to update entire resource i.e. whole object

PUT Request Example
app.put(“/path:”, (req, res) => {
req.body // used to get data to update
// code to update data storage
res.send()
})

PATCH request:

  • Used to update part of an object i.e. only one parameter

PATCH Request Example
app.patch(“/path”, (req, res) => {
req.body
//code to update resource
res.send()
})

DELETE request:

  • Used to delete resource/ all resources

DELETE Request Example
app.delete(“/path”, (req.res) => {
//code to delete resource
res.send()
})

Sending resource form .json or .html files

  • To get content of .json file:

    Example
    require(“./cat.json”) // best way
    require(“./cat”) // also works
    require(“cat”) // checks only node modules so it won't work
    })

  • To get requested HTML page:
    Use __dirname which returns path of the server, working with html.
    Name of the file storing HTML differs, it can be: static, www, templates, but in NodeJS is is usually called public

    Example
    res.sendFile(__dirname + “//folder name where HTML is stored>//page name.html”)
