Strict Mode

Allocate phrase: "use strict" to activate code execution in strict mode.
Declared at the beginning of a script (applied globally) or a function (applied only for function scope).
Strict mode is used to get clearer, secure code. i.e. strict mode when activated not allows to use:

  • Undeclared variables or objects

  • Deleting variables, objects or functions

  • Duplicating a parameter name

  • 'this' keyword refers to the object that called the function

  • 'eval()' is not allowed to create variables in the scope from which it was called

  • 'with' statement

  • The word arguments cannot be used as a variable

  • The word eval cannot be used as a variable

  • Deleting an undeletable property

  • Writing to a get-only property

  • Writing to a read-only property
